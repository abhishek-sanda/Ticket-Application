# üé´ Support Ticket Workflow

This is an internal tool built with Streamlit to demonstrate a simple support ticket management system. Users can create new tickets, view and edit existing ones, and see key statistics on a dashboard.

## ‚ú® Features

- **Create Tickets**: Add new support tickets by describing the issue and setting a priority level (High, Medium, Low).
- **View and Edit Tickets**: All existing tickets are displayed in an interactive table. Users can sort by any column and directly edit the `Status` and `Priority` of each ticket.
- **Persistent Session**: Ticket data, including new additions and edits, is maintained throughout the user's session.
- **Dashboard & Statistics**:
    - Key performance indicators (KPIs) are displayed using metrics, such as the total number of open tickets.
    - An interactive bar chart shows the number of tickets by status for each month.
    - An interactive pie chart visualizes the distribution of current ticket priorities.

## üõ†Ô∏è System Architecture

The application uses a simple client-server architecture orchestrated by the Streamlit framework. The backend logic, data processing, and UI definition all reside in a single Python script.

```text
+------------------------+
|      User's Browser    |
|     (Client-Side)      |
|   - Renders HTML/JS    |
|   - Captures Events    |
+-----------^------------+
            |
 (HTTP/WebSocket Communication)
            |
+-----------v------------+
|  Streamlit Server      |
|  (Runs ticket.py)      |
| +--------------------+ |
| |   App Logic        | |
| | (Python Script)    | |
| +--------------------+ |
|           |            |
| +---------v----------+ |
| | st.session_state   | |
| | (In-Memory Store)  | |
| | +----------------+ | |
| | | Pandas         | | |
| | | DataFrame      | | |
| | +----------------+ | |
| +--------------------+ |
+------------------------+
```

-   **Client-Side (User's Browser)**: Renders the HTML and JavaScript generated by Streamlit. It captures user interactions (e.g., form submissions, cell edits) and sends them to the server.
-   **Server-Side (Streamlit Application)**: A Python process that executes `ticket.py` from top to bottom in response to user interactions.
    -   **State Management**: `st.session_state` is used to maintain the ticket DataFrame in memory, ensuring data persists throughout a user's session.
    -   **Data Processing**: The `pandas` library manages all data manipulation, from creating the initial dataset to appending new tickets.
    -   **UI Generation**: Streamlit commands (`st.title`, `st.data_editor`, etc.) generate the UI components that are sent to the client for rendering.

## üöÄ Getting Started

To run this application on your local machine, follow these steps.

1.  **Clone the repository:**
    ```shell
    git clone https://github.com/abhishek-sanda/Ticket-Application.git
    cd Ticket
    ```

2.  **Create and activate a virtual environment:**
    ```shell
    # For Windows
    python -m venv venv
    .\venv\Scripts\activate
    ```

3.  **Install the dependencies:**
    ```shell
    pip install -r requirements.txt
    ```

4.  **Run the Streamlit app:**
    ```shell
    streamlit run ticket.py
    ```
